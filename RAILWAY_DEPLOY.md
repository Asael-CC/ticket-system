# Ticket System - Deployment Guide

## Railway Deployment (Recommended)

This project is configured for easy deployment on Railway platform.

### Files Created for Deployment

1. **Dockerfile** - Multi-stage build for production
2. **railway.json** - Railway configuration
3. **.env.example** - Environment variables template
4. **DEPLOY.md** - Complete deployment guide
5. **.dockerignore** - Excludes unnecessary files from Docker build
6. **nginx.conf** - Nginx configuration for frontend

### Quick Deploy Steps

1. Push code to GitHub
2. Create Railway project from GitHub repo
3. Add PostgreSQL database
4. Set environment variables
5. Deploy!

### Environment Variables Required

```bash
DATABASE_URL="postgresql://..."  # Auto-generated by Railway
JWT_SECRET="your-secret-key"     # Generate with: openssl rand -base64 32
NODE_ENV="production"
PORT=3001
```

### Frontend Deployment Options

**Option 1: Netlify (Recommended)**
- Free tier available
- Automatic HTTPS
- Easy GitHub integration

**Option 2: Railway**
- Same platform as backend
- Shared project management
- Good for testing

### Architecture in Production

```
User → Frontend (Netlify/Railway) → Backend API (Railway) → PostgreSQL (Railway)
```
